## 2. 分层架构（与 moduleAccess 不冲突）

```
Layer 0  部署与信道：DEPLOYMENT_ROLE × channel(local|lan|public)
Layer 1  moduleAccess：hidden / requireLogin（全局，仅超管配）
Layer 2  身份：JWT；is_super_admin
Layer 3  能力授权：catalog + grant + 作用域 + 再授权标志  ← 本文档
```

**写操作判定顺序：**

1. 模块是否可见？（Layer 1）
2. 是否要求登录且未登录？（Layer 1）
3. 平台超管？→ 放行
4. 持有所需 capability（含作用域）？
5. 业务规则（发布锁等）

公网只读接口（如 appStore 下载）可跳过 Layer 3。

## 3. 核心概念

### 3.1 主体与作用域

| 概念 | 说明 |
|------|------|
| Subject | 用户 `user_id` |
| Scope | `org` / `project`（平台级能力仅超管隐式持有，不入库） |
| Capability | 稳定字符串，如 `log.key.create` |
| Grant | 用户在某作用域持有某能力 + `can_delegate` / `can_revoke_peer` |

- **org 级 grant** ⇒ 覆盖该 org 下所有 project。
- **project 级 grant** ⇒ 仅该 project；**不能**升格为 org 管理。
- 需要 org 作用域的 API：必须命中 org 级 grant（project 不够）。

### 3.2 租户管理员（展示名）

创建租户时：

1. 平台超管创建 `sys_org`。
2. 指定或创建用户 U。
3. 向 U 写入 `PRESET_TENANT_ADMIN`（org 作用域），`can_delegate=1`，`can_revoke_peer=1`，`grant_source='tenant_bootstrap'`。

之后无身份差异；禁止 `is_tenant_admin` 字段。

### 3.3 再授权 / 再收回

| 字段 | 含义 | 默认 |
|------|------|------|
| `can_delegate` | 能否把同一 capability + 同或更窄 scope 再授出 | `true` |
| `can_revoke_peer` | 能否收回他人在同 scope 的同一 capability | `false` |

| 操作 | 条件 |
|------|------|
| 授出 C 给 B | A 持有 C 且 `can_delegate`；授出标志 ⊆ A 的标志（不可抬权） |
| 收回自己发出的 | `granted_by = A`，或超管 |
| 收回他人同权 | A 持有 C 且 `can_revoke_peer`，或超管 |
| 放弃自己的 | 允许；写审计 |

### 3.4 平台超管与局域网

| 身份 | 行为 |
|------|------|
| 已登录超管 | 跳过一切 capability；可改 moduleAccess；可建租户 |
| 局域网未登录 | 权限类 API 一律 401；禁止匿名当超管 |
| 已登录非超管 | 仅 Layer 3 |

### 6.2 核心逻辑

```js
export function matchScope(grant, needType, needId, projectOrgId) {
  if (grant.scope_type === 'org' && grant.scope_id === 0 && needType === 'platform') {
    return true; // 模块级 platform 能力
  }
  if (needType === 'org') {
    return grant.scope_type === 'org' && Number(grant.scope_id) === Number(needId);
  }
  if (needType === 'project') {
    if (grant.scope_type === 'project' && Number(grant.scope_id) === Number(needId)) return true;
    if (grant.scope_type === 'org' && projectOrgId != null
        && Number(grant.scope_id) === Number(projectOrgId)) return true;
  }
  return false;
}

export function hasCapability(user, capability, scope, grants, projectOrgId) {
  if (!user) return false;
  if (user.is_super_admin) return true;
  return grants.some(g =>
    g.capability === capability && matchScope(g, scope.scopeType, scope.scopeId, projectOrgId)
  );
}

export function clampFlags(operatorGrant, requested = {}) {
  return {
    can_delegate: !!(requested.canDelegate !== false && operatorGrant.can_delegate),
    can_revoke_peer: !!(requested.canRevokePeer && operatorGrant.can_revoke_peer),
  };
}
```
