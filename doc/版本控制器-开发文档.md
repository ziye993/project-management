# 应用商店（appStore）开发文档 — AI 实现说明

> 本文档供 AI / 开发者按现有仓库规范实现「应用商店」模块。  
> 决策已定稿；未标明「二期」的内容均为首版必做。  
> 依据：`开发规范.md`；模板模块：`calc`（JSON CRUD）、`ChunkUploader`（大文件）、`proxyStatic`（静态下发）、`accessRules`（权限）。

## 0. 已定决策（勿再询问）

| 项 | 定稿 |
|----|------|
| 模块展示名 | 应用商店 |
| moduleKey | `appStore` |
| 路由 | `/app-store` |
| 公网 | **开放**：下载无鉴权；管理端可读；**写操作必须登录且具备本模块写权限** |
| 版本号下限 | 禁止 `0.0.0.0`；最低 `0.0.0.1` |

| 建议下一版本 | 取最新四元组，BUILD+1，向高位进位；比较规则见 §3.3 |
| 发布锁 | HTTP 租约；**必须登录**；无 Socket.IO |
| 心跳 | `lockHeartbeatMs = 20000`；`lockTtlMs = 40000`（≈2×心跳，避免临界过期；语义为无心跳约数十秒内释放） |
| 包大小上限 | 2GB |
| 同人多标签 | 最简单：同一 `userId` 可重新 `acquire` 覆盖自己的锁；不引入复杂接管 UI |
| 下载形态 | **按原始上传文件直接下载**（保留原扩展名与 `originalName`）；不强制打成 zip；勿二次打包 |
| 历史版本保留 | 最多保留 **10** 个最近版本，**额外**再保留最多 **2** 个大版本（形如 `X.Y.0.0`，无后缀） |
| 发布审计 | 写入 log 模块：`content` = 发布信息 JSON 字符串 |
| 公网更新链接 | `GET /appStore/:ownerSlug/:appSlug` → 该应用**最新已发布**包文件流 |

## 1. 目标

统一管理多应用的版本包与说明，提供类似商店的浏览/发布体验；客户端（Linux/Windows 等）可通过固定 HTTPS 链接拉取最新源文件/安装文件。

### 1.1 首版必做

- 应用列表（进模块先看列表）；Layout 操作区提供「新增」按钮
- 应用 CRUD：名称、`ownerSlug`、`appSlug`、封面、简介、状态
- 版本发布：建议版本号、数字输入改版本、changelog、上传原文件、发布锁冲突提示
- 公网无鉴权下载链接（最新版）
- 版本列表下载历史版（管理端）；历史自动裁剪
- 发布成功写 log（`content` JSON）
- 权限：写需登录 + `canWriteModule('appStore')`；下载公开
- 抽离可复用列表筛选组件，并改图像 / 影视（及为局域网共享预留接入点）

### 1.2 首版不做（二期）

- Git 分支真实绑定、CI 推送
- 强制 zip 包装、多平台同一版本多产物矩阵（若以后要做：同一 version 下 `artifacts[]`）
- Socket.IO 实时锁
- 草稿 `draft` 状态（首版直接 `published`）
- `?version=` 指定版本下载（可预留，非必须）

## 3. 领域模型

### 3.1 关系

```
App
 ├── ownerSlug, appSlug   → 公网路径 /appStore/{ownerSlug}/{appSlug}
 ├── name, cover, description, status
 ├── publishLock | null
 └── versions: { [version]: Version }

Version
 ├── version, title, changelog, branch?
 ├── status: published | yanked
 ├── file: { originalName, storedName, relativePath, size, mime, sha256? }
 └── createdAt, createdBy
```

### 3.2 `data/app-store.json` 结构

```json
{
  "schemaVersion": 1,
  "apps": {
    "<appId>": {
      "id": "uuid",
      "name": "AGV 客户端",
      "ownerSlug": "qingshan",
      "appSlug": "agv",
      "coverPath": "",
      "description": "",
      "status": "active",
      "createdAt": 0,
      "updatedAt": 0,
      "createdBy": { "userId": 1, "username": "xxx" },
      "latestVersion": "0.0.0.1",
      "publishLock": null,
      "versions": {}
    }
  }
}
```

约束：

- `(ownerSlug, appSlug)` 全局唯一；仅允许 `[a-z0-9][a-z0-9-]{0,63}`（小写）。
- `latestVersion`：状态为 `published` 的最大版本；无则 `null`。
- `publishLock`：

```json
{
  "userId": 1,
  "username": "zhangsan",
  "token": "random",
  "expiresAt": 0
}
```

### 3.3 版本号

**格式**：`MAJOR.MINOR.PATCH.BUILD[-SUFFIX]`

- 每段整数 `0..999`
- 禁止四段全 0（`0.0.0.0`，无论有无后缀）
- 最低合法：`0.0.0.1`
- SUFFIX 可选：`[A-Za-z0-9][A-Za-z0-9._-]{0,31}`

**正则**（前后端各实现一份，逻辑一致）：

```text
^(?:0|[1-9]\d{0,2})\.(?:0|[1-9]\d{0,2})\.(?:0|[1-9]\d{0,2})\.(?:0|[1-9]\d{0,2})(?:-[A-Za-z0-9][A-Za-z0-9._-]{0,31})?$
```

另加：解析后若四段均为 0 → 非法。

**比较**：

1. 比 MAJOR→MINOR→PATCH→BUILD  
2. 四段相同：无后缀 > 有后缀  
3. 仍相同：比 `createdAt`

**建议下一版本** `suggestNext(app)`：

1. 在未 yank 的 `published` 版本中取比较最大的四元组  
2. 无历史 → 返回 `0.0.0.1`  
3. 否则 BUILD+1；BUILD>999 则 PATCH+1 且 BUILD=0；依次向 MINOR、MAJOR 进位；MAJOR>999 → 错误「版本号已达上限」  
4. 建议值**不带**后缀

**大版本判定**（保留策略用）：无后缀且 `PATCH===0 && BUILD===0`，如 `0.3.0.0`、`0.2.0.0`、`1.0.0.0`。

文件：

- FE: `web/src/page/AppStore/utils/version.ts`
- BE: `server/src/server/appStore/version.js`

### 3.4 历史版本裁剪

每次 `publish` 成功后执行 `pruneVersions(app)`：

1. 全部非 yank 版本按比较序排序（新→旧）  
2. `keep` = 最近 **10** 条  
3. 从全部版本中筛「大版本」，按比较序取最高 **2** 条，并入 `keep`  
4. 不在 `keep` 中的版本：删元数据 + 删磁盘文件  
5. 更新 `latestVersion`

yank 版本不进入 `keep`，裁剪时删除元数据与文件。

## 4. 公网拉取链接

### 4.1 URL

```
https://{host}:{port}/appStore/{ownerSlug}/{appSlug}
```

例：`https://ziye993.cn:40014/appStore/qingshan/agv`

行为：

- `GET`，**无需登录**，任意 channel  
- 解析 `ownerSlug`+`appSlug` → App → `latestVersion` 对应 `published` 文件  
- 响应：文件流；`Content-Type` 按 mime 或 `application/octet-stream`  
- `Content-Disposition: attachment; filename="<originalName>"`（注意 `filename*` UTF-8 编码）  
- 无应用 / 无版本 / 非 active：`404`

### 4.2 实现位置

`server/src/server/appStore/download.js` 注册：

```js
app.get('/appStore/:ownerSlug/:appSlug', handler);
```

约束：

- 在 SPA fallback **之前**注册，避免被前端路由吞掉  
- 路径规范化，禁止 `..`  
- 只允许读元数据中的 `relativePath`

管理端详情展示「更新链接」：用 `PUBLIC_BASE_URL` / 现有 accessLinks 同类逻辑拼绝对 URL，可复制。

### 4.3 原始文件下载（非 zip）

- 上传什么扩展名，磁盘与下载就用什么；**禁止**发布后再包一层 zip  
- `originalName` 作为下载文件名  
- 客户端 curl / wget / 浏览器均可

## 5. 发布锁（HTTP 租约）

### 5.1 规则

- **仅登录用户**可 `acquire` / `heartbeat` / `release` / `publish`  
- `userId`、`username` 取自 `req.user`；以服务端会话为准，不信任客户端伪造身份  
- `token` 服务端生成；后续操作必须带 `lockToken`  
- `lockHeartbeatMs=20000`，`lockTtlMs=40000`  
- 同 `userId` 再次 `acquire`：覆盖旧锁并颁发新 token（多标签最简策略）  
- 他用户持锁且未过期：`code=40901`，返回 `username`、`expiresAt`  
- 过期锁视为无锁  

### 5.2 前端

- 进入发布流程先 `lock/acquire`  
- 冲突文案：「该应用正在由「{username}」发布中，请协商；对方退出或锁超时后可继续。」  
- 每 20s `heartbeat` 
- 卸载 / 关闭发布 / publish 成功 → `release`；`beforeunload` 可用 `sendBeacon`（失败则依赖 TTL）

## 6. 存储与上传

### 6.1 目录

```
data/files/appStore/
  covers/{appId}/...
  packages/{appId}/{version}/{storedName}
```

`initDataStorage` 创建；config 默认：

```json
{
  "appStorePackagePath": "<data>/files/appStore",
  "appStoreRequestPath": "/static/app-store",
  "appStore": {
    "lockTtlMs": 40000,
    "lockHeartbeatMs": 20000,
    "maxPackageBytes": 2147483648,
    "maxHistoryVersions": 10,
    "maxMajorVersions": 2,
    "allowedExts": []
  }
}
```

`allowedExts` 空 = 不限制扩展名。

### 6.2 上传流程（MVP）

1. 登录用户持锁  
2. `ChunkUploader` → `chunkInit/chunk/chunkMerge`（复用现有 file 类型）  
3. `version/publish`：校验锁与版本号 → 临时文件 **move** 到 `packages/{appId}/{version}/` → 写元数据 → prune → 写 log → release  

封面可选。单文件上限 2GB。

## 7. 权限

| 操作 | 条件 |
|------|------|
| `GET /appStore/...` 下载 | 无限制 |
| 浏览应用/版本列表 API | 模块可见；公网 **允许读** |
| 新增/编辑应用、发布、抢锁、改说明、yank | **登录** + 模块写权限 |

实现要点：

- capabilities：`appStore` 在 `public` 下 `read: true`；`write` 仅已登录（超管全开）  
- **不要**把 `appStore` 放进 `PUBLIC_NEVER`  
- 写 API：无 `req.user` → 401  
- 首版发布以局域网/本机登录为主；**公网必须保证匿名可下载**。若公网登录后也要发布，再评估对已登录用户放行 chunk，且不得放开匿名媒体写。

## 8. API

除拉取为 `GET` 外，均为 `POST` + JSON。响应：`ok` / `fail`。

### 8.1 应用

| 路径 | 鉴权 | Body |
|------|------|------|
| `/api/appStore/app/list` | 读 | `{ keyword? }` |
| `/api/appStore/app/get` | 读 | `{ appId }` |
| `/api/appStore/app/save` | 写+登录 | `{ id?, name, ownerSlug, appSlug, description, coverPath?, status? }` |
| `/api/appStore/app/delete` | 写+登录 | `{ appId }`（建议软删 `archived`） |

### 8.2 版本

| 路径 | 鉴权 | Body |
|------|------|------|
| `/api/appStore/version/list` | 读 | `{ appId }` |
| `/api/appStore/version/get` | 读 | `{ appId, version }` |
| `/api/appStore/version/suggest` | 读 | `{ appId }` → `{ version }` |
| `/api/appStore/version/publish` | 写+登录+锁 | 见下 |
| `/api/appStore/version/updateMeta` | 写+登录 | `{ appId, version, title?, changelog? }` |
| `/api/appStore/version/yank` | 写+登录 | `{ appId, version }` |

publish body：

```json
{
  "appId": "",
  "lockToken": "",
  "version": "0.0.0.2",
  "title": "",
  "changelog": "",
  "branch": "",
  "tempFile": {
    "storedName": "",
    "relativePath": "",
    "originalName": "",
    "size": 0,
    "mime": ""
  }

```

### 8.3 锁

| 路径 | 鉴权 |
|------|------|
| `/api/appStore/lock/acquire` | 登录+写 `{ appId }` |
| `/api/appStore/lock/heartbeat` | 登录 `{ appId, lockToken }` |
| `/api/appStore/lock/release` | 登录 `{ appId, lockToken }` |
| `/api/appStore/lock/status` | 读 `{ appId }` |

冲突：`code=40901`。

## 14. 验收标准

- [ ] 首页有「应用商店」；进入为应用列表；写权限可见「新增」  
- [ ] 应用 `qingshan`/`agv` 发布后，`GET /appStore/qingshan/agv` 下载为**原始文件**，非强制 zip  
- [ ] 未登录可下载；未登录不能 publish/acquire  
- [ ] 最新 `0.0.0.1` 时发布页四段为 `0,0,0,2`  
- [ ] 无法发布 `0.0.0.0`  
- [ ] A 持锁时 B 见冲突文案；A release 或 TTL 后 B 可发布  
- [ ] 同用户第二标签 acquire 成功  
- [ ] 裁剪后 ≤10 近期 + ≤2 个 `X.Y.0.0` 大版本，磁盘同步删  
- [ ] 发布后 log `content` 为约定 JSON（含 uploader、ip、fileName、fileSize、version）  
- [ ] 图像/影视使用通用 Filter；聊天 toggle 仍可用；支持按名搜索  
- [ ] `appStore` 不在 `PUBLIC_NEVER`；公网可读；匿名不可写

## 15. 文案

| 场景 | 文案 |
|------|------|
| 锁冲突 | 该应用正在由「{username}」发布中，请协商；对方退出或锁超时后可继续。 |
| 版本已存在 | 版本 {ver} 已存在，请更换版本号。 |
| 非法版本 | 版本号须为 0~999 四段，可选 -特殊值；不能为 0.0.0.0，最低 0.0.0.1。 |
| 锁失效 | 发布会话已失效，请重新进入发布。 |
| 文件过大 | 文件超过 2GB 上限。 |
